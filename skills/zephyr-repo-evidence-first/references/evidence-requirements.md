# 证据最低要求

所有关于当前项目的结论必须附带证据。

| 结论 | 最少证据 |
| --- | --- |
| 当前实现方式 | 文件路径 + 类 / 函数 / 配置项 |
| 某依赖是否实际使用 | 构建依赖 + 配置或调用点 |
| 某模块是否存在 | 模块路径 + 实现或调用点 |
| 某接口是否已存在 | 路由 / Controller / API client / 测试 |
| 某字段是否已存在 | 类型、实体、DTO、Schema、迁移或查询 |
| 某行为是否生效 | 调用链、测试、配置或运行逻辑 |
| 应复用的模块 | 具体文件、类、函数或组件 |
| 应新增的内容 | 已确认缺口与受影响范围 |

仅发现依赖、表名、配置项或目录，不足以证明功能已启用或正在使用。

## 调用链要求

调用链至少还原为：

```text
调用方 → 入口 → 业务逻辑 → 数据 / 外部服务 → 返回或副作用
```

前后端任务还应确认：

```text
页面 / 组件 → API client → 后端入口 → 业务层 → 数据层
```
