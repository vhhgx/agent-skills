# 前端结论证据要求

前端结论至少应定位到：文件路径 + 组件 / Hook / Store / 函数 / 类型 / 路由 / 配置项。

| 结论 | 最少证据 |
| --- | --- |
| 页面是否存在 | 路由配置 + 页面组件 |
| 页面入口在哪里 | 路由、菜单配置、跳转调用点或导航组件 |
| 某交互是否已实现 | 组件事件处理函数 + 对应状态或 API 调用 |
| 某接口是否被前端调用 | API client / service + 调用组件、hook 或 store |
| 某字段是否被前端依赖 | TypeScript 类型、Schema、组件渲染、表单字段或状态逻辑 |
| 某接口返回结构是否可改 | API 类型定义 + 所有调用点 + 数据转换逻辑 |
| 某数据是否来自 Mock | Mock 文件 + Mock 启用配置 + 调用链 |
| 某状态由哪里维护 | Store、Context、Hook、URL Query、组件 State 或缓存逻辑 |
| 某权限控制是否存在 | 路由守卫、菜单过滤、按钮权限指令、组件条件渲染或权限 Hook |
| 某页面是否可复用已有组件 | 已有组件路径 + Props 定义 + 实际使用场景 |
| 某前端配置是否生效 | 环境变量、构建配置、运行时配置 + 实际读取位置 |
| 某功能是否已有测试 | 测试文件 + 对应页面、组件、Hook 或 API mock |

以下任一项单独存在时，不足以得出前端功能已实现的结论：

- 只有页面文件，没有路由或入口。
- 只有 API client，没有调用点。
- 只有 TypeScript 类型，没有真实渲染或数据消费逻辑。
- 只有 Mock 数据，没有 Mock 启用配置或调用链。
- 只有权限字段，没有路由、菜单或按钮控制逻辑。
- 只有组件文件，没有实际引用位置。
- 只有环境变量定义，没有读取和生效位置。
