# 禁止行为与自检

## 禁止行为

禁止：

- 未检索代码就回答“当前项目应该如何实现”。
- 根据依赖名称推断功能已启用。
- 根据表名、目录名或模块名推断实际行为。
- 根据技术栈生成通用脚手架。
- 未确认就新建平行模块、接口、数据模型或配置。
- 未确认就修改既有字段、返回结构、错误码、事件名或状态值。
- 忽略已有同类实现。
- 只看单个文件就概括整个系统。
- 将推测、惯例或最佳实践表述为项目事实。
- 用“如果……那么……”替代仓库分析。
- 在没有证据摘要前直接开始编码。

## 条件式猜测处理

禁止使用以下表达替代结论：

- 如果项目已经有……
- 如果使用了……
- 通常需要……
- 一般会有……
- 大概率……
- 可以考虑……
- 建议新建……
- 可能存在……
- 一般应该……

只有在工作区无法确认、且确实需要外部信息时，写入待确认项。

## 自检

- 是否已围绕当前任务读取相关代码、配置、调用链、测试和同类实现。
- 是否所有当前项目结论都有可定位证据。
- 是否把未确认项与已确认结论分离。
- 是否明确区分复用、补齐和新增。
- 是否避免基于惯例生成平行实现。
- 是否在实施前说明调用链、影响范围和验证方式。
- 是否完成必要验证并记录命令与结果。
