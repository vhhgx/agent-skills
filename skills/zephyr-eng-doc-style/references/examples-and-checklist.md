# 示例与自检

## 规则压缩

原文：

前端权限控制只用于用户体验，后端必须校验：登录态、当前数据范围、资源归属、角色权限、敏感操作审计。

改为：

后端不得信任前端权限判断；所有业务接口必须校验认证、数据范围、资源归属、权限和审计。

## 页面要求压缩

原文：

建议页面顶部或者用户菜单显示当前租户：当前租户、当前角色。

改为：

页面顶部或用户菜单必须显示当前租户和当前角色。

## 字段说明结构化

原文：

账号列表需要展示生命周期状态、连接状态、登录方式、绑定代理、最近在线时间、最近离线时间、最近掉线原因、今日发送数、今日失败数、归属人和所属租户。

改为：

| 字段 | 说明 |
| --- | --- |
| `lifecycleStatus` | 生命周期状态 |
| `connectionStatus` | 连接状态 |
| `loginMethod` | 登录方式 |
| `proxyName` | 绑定代理 |
| `lastOnlineAt` | 最近在线时间 |
| `lastOfflineAt` | 最近离线时间 |
| `lastOfflineReason` | 最近掉线原因 |
| `todaySentCount` | 今日发送数 |
| `todayFailedCount` | 今日失败数 |
| `ownerName` | 归属人 |
| `tenantName` | 所属租户 |

## 待确认项隔离

错误：

建议后续考虑增加自动重试，也许需要增加重试次数配置。

正确：

| 待确认项 | 影响 | 默认处理 |
| --- | --- | --- |
| 是否支持自动重试 | 任务失败处理与配置设计 | 本期不新增自动重试 |

## 生成前自检

- 是否保留接口路径、权限字符、状态值、字段名和错误码等定义。
- 是否错误截断权限字符、状态值、路径或字段名。
- 是否将普通列表放入代码块。
- 是否有超过 8 项的短列表可改为表格。
- 是否有状态流转仍在使用流水账。
- 是否有映射关系仍在使用长段说明。
- 是否有解释性段落可压缩为规则、表格或一行结论。
- 是否存在“建议、相关、一些、等等、通常、一般”等模糊词。
- 是否将待确认项和已确认结论分开。
- 是否有重复的标题、背景或说明。
- 是否把大段 Schema、JSON、SQL 或日志塞入正文。
- 是否让人类能够快速定位定义、规则、影响和待确认项。
- 是否让后续 Agent 能准确提取关键约束，而无需猜测。
