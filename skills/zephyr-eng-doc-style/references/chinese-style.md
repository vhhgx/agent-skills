# 中文表达与篇幅控制

## 句子规则

1. 一句话尽量只表达一个事实。
2. 主语、动作、结果应清楚。
3. 优先使用主动、直接、可执行的动词。
4. 少用抽象名词堆砌。
5. 避免句子同时描述背景、规则、原因和结果。
6. 不使用口语化、宣传式或培训式表达。
7. 已确认事实使用陈述句；不确定内容使用“待确认”。

优先使用的动词：校验、查询、返回、创建、更新、删除、过滤、记录、阻断、跳过、转换、聚合、重试、终止、复用、迁移、废弃。

## 空泛表达替换

| 避免使用 | 推荐替换 |
| --- | --- |
| 进行 | 直接使用动作动词 |
| 实现相关能力 | 明确实现对象和行为 |
| 支持 | 写明具体可执行行为 |
| 相关 | 明确关联对象 |
| 一些 | 列出具体对象或删除 |
| 等等 | 列尽或写“待补充” |
| 若干 | 写明确数量或范围 |
| 主要用于 | 直接描述行为与结果 |
| 可以对……进行 | 直接描述动作 |
| 为了……需要…… | 直接陈述规则或约束 |

## 模糊词处理

以下词语不得直接作为需求、规则或结论：建议、最好、可以考虑、需要的话、可能、尽量、一般、通常、大概率、相关、一些、若干、等等。

| 情况 | 表达 |
| --- | --- |
| 强制要求 | 必须 |
| 可选能力 | 可选 |
| 当前范围 | 本期实现 |
| 延后处理 | 后续实现 |
| 缺少事实 | 待确认 |
| 明确排除 | 非本期范围 |
| 方案选择 | 方案 A / 方案 B，并说明差异 |

“建议”只允许出现在明确标注为“建议方案”的非强制性设计讨论中，且必须说明是否进入本期。

## 篇幅阈值

| 项目 | 默认阈值 | 超出后的处理 |
| --- | --- | --- |
| 背景 | 全文 10% | 压缩或移除 |
| 单个章节 | 30 行 | 按主题拆分 |
| 单个代码块 | 15 行 | 移入附录或独立文件 |
| 连续普通列表 | 8 项 | 优先改为表格或分组 |
| 单个表格单元格 | 2 句以内 | 拆分字段或改为补充说明 |

正文应保留足以支持审阅和决策的信息，不应变成源代码镜像。
