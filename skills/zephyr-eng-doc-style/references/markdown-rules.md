# Markdown 使用规则

## 代码块

代码块只用于需要精确保留格式或可复制执行的内容。

允许：SQL、JSON、YAML、Java / TypeScript / Python 等代码、OpenAPI、可执行 Shell 命令、配置片段、精确 Schema、正则表达式、复杂查询语句。

禁止将以下内容放入代码块：普通列表、权限字符说明、状态枚举说明、字段说明、页面字段、操作项、实施步骤、设计原则、结论、业务规则。

错误：

```text
账号状态：
PENDING_LOGIN 待登录
ONLINE 在线
OFFLINE 离线
```

正确：

| 状态 | 说明 |
| --- | --- |
| `PENDING_LOGIN` | 待登录 |
| `ONLINE` | 在线 |
| `OFFLINE` | 离线 |

## 列表

列表只用于项目之间不能自然构成表格、映射或短句的场景。

可以使用列表的情况：每项需要单独展开；每项有独立负责人、优先级、状态或截止条件；每项文字较长；表达顺序明确的操作步骤；列表项目之间不是固定字段关系。

不应使用列表的情况：只是枚举多个校验项、字段、状态、权限码、页面能力或短操作项。

错误：

需要校验：

- 权限
- 数据范围
- 频控
- 审计

正确：

发送前校验：权限、数据范围、频控和审计。

## 表格

以下关系优先使用表格：字段与说明、权限字符与含义、状态与含义、状态迁移、接口与用途、页面与影响、实体与关系、条件与处理结果、任务与依赖、风险与处理方式、待确认项与影响。

表格字段必须短、稳定、可比较。避免把整段解释塞进单元格。

## 标题

标题只用于分隔独立主题。禁止一个标题下只有一句空泛说明、连续多级标题但没有结构性内容、用“目标/说明/注意事项”重复包装同一事实、为每个短句创建标题。

标题下应直接出现结论、定义、规则、表格、状态迁移、映射关系、实施项或待确认项。
