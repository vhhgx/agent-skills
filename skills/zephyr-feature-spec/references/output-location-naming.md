# 文档输出位置与命名

## 文件名

功能规格或计划文档必须使用：

```text
MMDD-序号-中文描述.md
```

| 项 | 规则 |
| --- | --- |
| 日期 | 使用创建当天月日；6 月 25 日写为 `0625` |
| 序号 | 在目标目录中查找同日期文档，使用最大序号加一 |
| 初始序号 | 当天没有同类文件时使用 `01` |
| 描述 | 使用简短、可识别的中文功能名 |
| 描述限制 | 不使用“方案”“文档”“计划”“功能”等无信息后缀 |
| 扩展名 | `.md` |

示例：

```text
0625-01-账号生命周期.md
0625-02-批量消息任务.md
0625-03-联系人标签管理.md
```

## 输出目录解析

生成文档前，必须读取项目规范和现有目录结构，并按以下优先级确定路径：

| 优先级 | 判断条件 | 输出路径 |
| --- | --- | --- |
| 1 | 用户明确指定路径 | 使用用户指定路径 |
| 2 | `AGENTS.md`、`CLAUDE.md`、README 或项目规范明确指定文档目录 | 使用规范指定目录 |
| 3 | 项目已有同类功能规格或计划目录 | 沿用现有目录与命名习惯 |
| 4 | 存在 `docs/plan/` | 使用 `docs/plan/` |
| 5 | 存在 `docs/specs/` | 使用 `docs/specs/` |
| 6 | 以上均不存在 | 创建并使用 `docs/plan/` |

生成前必须检查目标目录是否存在、当天已有文件最大序号、避免覆盖已有文件，并说明最终路径和路径依据。

如仓库存在多套候选目录，且无法从规范或已有文件判断用途，必须列为待确认项；不得自行写入不确定目录。
