# 功能规格文档结构

生成的文档必须按以下结构输出：

```text
# <功能名称>

## 1. 结论
## 2. 当前现状
## 3. 本期范围
## 4. 核心规则
## 5. 前端影响
## 6. 后端影响
## 7. 数据影响
## 8. 接口与交互边界
## 9. 实施计划
## 10. 验收标准
## 11. 待确认项
```

## 结论

最多 5 行，必须说明本次功能目标、本期交付边界、最关键的实现约束、文档基于哪些当前工作区事实形成。

禁止写背景故事、行业说明或泛化目标。

## 当前现状

必须包含证据表。不描述与当前任务无关的模块。

| --- | --- | --- |
| 入口 |  |  |

| 数据来源 |  |  |
| 同类能力 |  |  |
| 当前限制 |  |  |
| 已有测试 |  |  |

## 本期范围

必须同时写“本期实现”和“非本期范围”。没有明确非本期范围时，写：非本期范围：未发现明确排除项。

| 模块 | 本期实现 | 非本期范围 |
| --- | --- | --- |
| 前端 |  |  |
| 后端 |  |  |
| 数据库 |  |  |
| 配置/部署 |  |  |
| 测试 |  |  |

## 待确认项

| 待确认项 | 影响 | 当前证据 | 默认处理 |
| --- | --- | --- | --- |
|  |  |  | 不作为实现前提 |

待确认项必须与已确认结论隔离。
