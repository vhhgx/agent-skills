# 后端任务、联调验收与自检

## 非接口能力交接详情

接口以外的能力单独交接，不强行伪装为 HTTP 接口。适用类型：WebSocket 事件、消息队列事件、定时任务、异步任务、文件上传流程、状态同步、配置项、前端修正项。

| 项 | 内容 |
| --- | --- |
| 状态 | 复用 / 修改 / 新增 / 前端修正 / 待确认 |
| 类型 | WebSocket / 事件 / 异步任务 / 配置项 / 其他 |
| 前端调用或依赖位置 | 文件路径 + 组件、Hook、函数或 Store |
| 前端依赖行为 | 前端实际等待、消费或触发的行为 |
| 后端现状 | 当前实现、行为缺口或未发现结果 |
| 优先级 | P0 / P1 / P2 |
| 完成条件 | 后端完成后前端可验证的结果 |
| 待确认项 | 当前无法确认的实现边界 |

任务必须明确对象和结果。禁止使用“开发后端”“补齐接口”“处理权限”“增加测试”。

正确表达：

| 优先级 | 任务 | 类型 | 完成条件 |
| --- | --- | --- | --- |
| P0 | 新增账号重连入口并调用现有连接服务 | 新增接口 | 前端可发起重连并收到状态结果 |
| P0 | 扩展账号列表返回生命周期状态字段 | 修改接口 | 列表可按状态渲染与筛选 |
| P1 | 替换列表页 Mock 数据源 | 前端联调 | Mock 关闭后页面正常加载 |

## 生成前自检

- 是否取得当前任务相关的前端与后端工作区分析结果。
- 是否检查 Git Diff，或明确记录无法检查的原因。
- 是否每项接口都有前端调用位置和后端核对证据。
- 是否 Method、Path、参数、字段和 Mock 来源均有证据。
- 是否将“未发现”与“确定不存在”区分。
- 是否将前端 Mock 与后端事实分离。
- 是否只列出前端实际使用或已确认需求明确要求的输入和输出字段。
- 是否权限、租户、资源归属和错误处理均有项目证据或已确认需求。
- 是否无证据项已标记为待确认，而不是套用通用模板。
- 是否每个新增或修改接口都有独立的后端开发 Checklist。
- 是否 Mock 差异已说明切换条件。
- 是否任务按解除 Mock 和主流程联调的依赖顺序排列。
- 是否避免展开全系统 API 治理、数据库设计或架构设计。
- 是否待确认项没有混入已确认结论。
- 是否遵循本文的文档表达和命名要求。
