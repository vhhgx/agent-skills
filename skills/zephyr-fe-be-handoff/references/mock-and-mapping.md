# Mock 与映射规则

## 当前 Mock 数据来源

| 项 | 内容 |
| --- | --- |
| Mock 文件 | 文件路径 |
| Mock 位置 | handler、fixture、函数、常量或硬编码位置 |
| 启用方式 | 环境变量、配置项、拦截器、条件分支或调用入口 |
| 当前行为 | 当前 Mock 模拟的返回或副作用 |
| 与真实后端差异 | 字段、状态、分页、错误处理、异步行为或权限差异 |
| 切换条件 | 后端提供哪些能力后可关闭或替换 Mock |

不复制大段 Mock JSON。

## Mock 与真实能力差异

| 前端 Mock 或临时行为 | 启用位置 | 后端当前能力 | 差异 | 处理结论 |
| --- | --- | --- | --- | --- |
|  |  |  |  | 新增 / 修改 / 前端修正 / 待确认 |

禁止把 Mock 中出现的所有字段默认视为后端必须返回的字段。字段是否需要由前端实际读取、类型约束、业务流程或已确认需求共同判断。

## 状态与枚举

当前接口涉及状态值、枚举值或操作约束时，必须单独列出。

| 字段 | 枚举值 | 含义 | 前端行为 | 来源 |
| --- | --- | --- | --- | --- |
| `lifecycleStatus` | `PENDING_LOGIN` | 待登录 | 显示登录入口 | Mock / 类型 / 页面判断 |
| `lifecycleStatus` | `ONLINE` | 在线 | 禁用登录入口 | Mock / 类型 / 页面判断 |

无法确认完整枚举时，不得补全枚举，只保留已发现值，并标记待确认。
