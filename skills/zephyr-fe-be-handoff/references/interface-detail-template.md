# 接口交接详情模板

每个新增、修改、复用确认、前端修正或待确认的接口，必须单独输出接口交接详情。

禁止为补全文档而凭空补充参数、字段、错误码、权限或租户规则。

## 基本信息

### <接口名称>

| 项 | 内容 |
| --- | --- |
| 状态 | 复用 / 修改 / 新增 / 前端修正 / 待确认 |
| Method | `GET` / `POST` / `PUT` / `PATCH` / `DELETE` |
| Path | `/api/...` |
| 前端调用位置 | 文件路径 + 组件、Hook、函数、Store 或 service |
| 用途 | 前端通过该接口完成的具体行为 |
| 优先级 | P0 / P1 / P2 |
| 后端现状 | 当前实现、字段缺口、行为缺口或未发现结果 |
| Mock 来源 | 文件路径 + handler、fixture、函数或硬编码位置 |
| 接口来源 | 前端调用 / Mock / 已有后端接口 / 已确认需求 |

接口名称、Method 和 Path 必须保留前端或后端已存在的原始定义。无法确认 Method 或 Path 时，标记为待确认。

## Request 示例

仅在前端 Mock、请求类型、Schema 或调用代码可证明完整结构时输出。无法确认完整结构时，不输出猜测性 JSON，只保留参数表。

## Response 示例

Response 示例只能由前端 Mock、fixture、MSW handler、TypeScript Response 类型、Schema、页面实际读取字段、已确认需求或已有后端响应拼出。

Response 示例仅用于表达前端实际依赖的返回结构，不代表全量接口协议。

## Response 字段

| 字段 | 类型 | 必填 | 含义 | 示例 | 枚举 | 前端用途 | 强依赖 | 证据 |

## 权限、租户与资源归属

仅记录当前项目代码、项目规范、前端权限逻辑、已有后端实现或已确认需求可证明的约束。

| 项 | 要求 | 来源 | 状态 |
| --- | --- | --- | --- |
| 权限 | 待确认 | 当前检索范围未发现接口权限或前端权限证据 | 待确认 |

未找到证据时，不得默认填写“需要权限校验”或“需要租户隔离”。

| 场景 | HTTP 状态 | 业务码 | 前端处理 | 来源 |
| --- | --- | --- | --- | --- |
| 参数错误 | `400` | `INVALID_PARAMS` | 展示表单错误 | Mock / 现有错误处理 |
| 当前状态不可操作 | 待确认 | 待确认 | 提示当前状态不可操作 | 页面状态判断 |

HTTP 状态、业务码或错误文案没有证据时，必须标记待确认。

## 后端开发 Checklist

每个新增或修改接口必须有独立 Checklist，仅保留当前接口适用项。

- [ ] 新增或修改 `Method + Path`
- [ ] 校验 Query、Path 和 Body 参数
- [ ] 按 Response 字段表返回前端强依赖字段
- [ ] 实现已确认的状态与枚举约束
- [ ] 实现已确认的权限、租户和资源归属约束
- [ ] 返回前端已处理或已确认的错误结果
- [ ] 覆盖当前接口的核心成功与失败测试
- [ ] 前端关闭或替换 Mock 后完成联调
